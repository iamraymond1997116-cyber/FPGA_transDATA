# 投稿预期评估

> 日期：2026-06-09 | 基于项目当前定位：「自定义 FPGA 测试平台 + 可编程电源激励框架 + 真实传感器数据」的全栈传感器身份认证工作。

---

## 一、核心定位（评价一切的基准）

本工作不是纯算法论文，是 **全栈硬件 + 系统 + 分析方法论文**：

```
自己的板子 → 自己的 Verilog → 5 种 MODE → 256 种 SCALE_SCH → 10 传感器真实数据 → 分析链
```

算法（SupCon/对比学习）是分析工具，不是核心创新。核心创新是 **FPGA 可编程激励框架用于传感器物理身份读取**。

对比基线：
- 纯算法组（Chen 2024, Jagannath 2022）跑公开数据集调参——与本工作不在同一赛道
- SRAM PUF 内建于芯片内——保护的是数字域安全，不解决传感器替换问题
- Arai 2026 继电器 PUF（*Sensors Actuators A*, IF~3.5）——机械瞬态，簧片老化快、信息量低

## 三、投稿策略建议

### 策略 A（推荐）：一步到位 Sensors J

**路线：** 补完 P0（SupCon + FAR/FRR + 消融）→ 投 Sensors J

**优点：**
- 审稿快（3-5 月），年内可见结果
- scope 天然匹配，不需要额外堆安全指标
- 中一篇后你简历上有发表，后续冲 TIFS 也有底气

**缺点：** IF 不如 TIFS

### 策略 B（冲刺）：先补实验再冲 TIM

**路线：** 补完 P0 → 架构描述重写 → **投 TIM (IF 5.3)**

**优点：** TIM 审稿快、IF 介于 Sensors J 和 TIFS 之间、硬件工作认可度高

**缺点：** 需要把论文重心从"安全范式"转到"测量系统与特征提取"上

### 策略 C（分阶段）：Sensors J → TIFS 扩展

**路线：** Sensors J 发"FPGA 传感器指纹测量系统 + 基线结果"→ 扩传感器量 + 攻击分析 + 开集检测 → TIFS

**可行性判断：**
- ⚠️ 两篇必须讲不同故事：
  - Sensors J：硬件系统论文（架构、实现、实测）
  - TIFS：安全分析论文（方法、攻击、安全性）
- ⚠️ 每篇必须有 >30% 新内容
- ✅ 如果 Sensors J 主要讲硬件基线，TIFS 主要讲对比学习 + 全面安全评估，这个路线可行

### 策略 D（保守）：TCAS-I

**路线：** 补少量实验 → 往电路架构角度重写 → 投 TCAS-I

**适合情况：** 想尽快发表且实验补充时间有限时选择
