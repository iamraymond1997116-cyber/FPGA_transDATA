# SRAM PUF 与传感器 PUF 差异化分析

> 日期：2026-06-10 | 聚焦技术差异，不涉及具体攻击案例归因

---

## 一、信任锚点位置

```
SRAM PUF:   [SRAM cell] ──→ 密钥 ──→ 信任根在 MCU/处理器
传感器 PUF: [传感器瞬态] → 密钥 ──→ 信任根在传感器
```

SRAM PUF 的信任锚点在数字域（芯片内部），传感器 PUF 把信任锚点**往下沉一层**——放到模拟域（物理传感元件）。这是最根本的架构差异，所有其他差异由此派生。

---

## 二、物理前提条件

| | SRAM PUF | 传感器 PUF |
|:---|:---|:---|
| 熵源 | SRAM 单元上电初始态 | 传感器上电/断电瞬态波形 |
| 是否需要芯片 | ✅ 需要硅基 SRAM | ❌ 仅需传感器能通电 |
| 覆盖无源传感器 | ❌ 热电偶/PT100/应变片无 SRAM | ✅ 导线+材料参数即熵源 |
| 覆盖物理分离的传感器 | ❌ SRAM 在 MCU 里，管不到线那头的传感器 | ✅ 瞬态特征通过线缆传回来 |

结论：传感器 PUF 的部署前提比 SRAM PUF 更弱——它不需要传感器里有数字电路。

---

## 三、输出能力对比

| 能力 | SRAM PUF | 传感器 PUF |
|:---|:---:|:---:|
| 身份认证（是/否） | ✅ | ✅ |
| 个体识别（具体是谁） | ✅ | ✅ |
| 型号/批次识别 | ❌ | ✅ |
| 传感器类型识别 | ❌ | ✅ |
| 健康状态评估 | ❌ | ✅（独有） |

SRAM PUF 输出一个 binary answer。传感器 PUF 单次采集输出四层信息：

```
输入: 128 点上电瞬态
输出: ├─ 传感器类型（B2 系列 vs 其他）
      ├─ 传感器子型号（B2-6 vs B2-8）
      ├─ 传感器个体身份（设备 ID）
      └─ 物理健康状态（退化分数）
```

**健康评估是 SRAM PUF 完全不具备的维度。** SRAM PUF 追求 startup pattern 越稳定越好——它刻意对物理状态变化不敏感。传感器 PUF 的瞬态特征天然随隔膜疲劳、接插件氧化、焊点微裂纹等物理退化而漂移——同样的信号，SRAM PUF 把它当噪声，传感器 PUF 把它当信息。

## 四、防护边界

| 威胁 | SRAM PUF | 传感器 PUF |
|:---|:---:|:---:|
| MCU/芯片替换 | ✅ | ❌ |
| 固件克隆 | ✅ | ❌ |
| 物理传感器替换 | ❌ | ✅ |
| 假冒传感器（型号/批次伪造） | ❌ | ✅ |
| 传感器物理退化检测 | ❌ | ✅（独有） |
| 软件/协议层攻击 | ❌ | ❌ 两者都不覆盖 |
| 网络层攻击 | ❌ | ❌ 两者都不覆盖 |

**两者是互补关系，不是替代关系。**

- SRAM PUF 覆盖数字域——芯片和固件的身份完整性
- 传感器 PUF 覆盖模拟域边界——传感器物理身份和健康状态
- 软件/网络层攻击需要其他纵深防御手段

---

## 五、两者在系统安全架构中的位置

```
                    ┌──────────────────────────────────┐
                    │         纵深防御层次              │
                    ├──────────────────────────────────┤
   网络层           │  通信安全 (TLS/DTLS/HMAC+seq)     │ ← 传统网络安全
                    ├──────────────────────────────────┤
   处理器层         │  SRAM PUF —— 芯片/固件身份        │ ← 数字域信任根
                    ├──────────────────────────────────┤
   传感器层         │  传感器 PUF —— 传感器身份+健康     │ ← 模拟域信任根（本工作）
                    ├──────────────────────────────────┤
   物理层           │  传感器本身                        │ ← 熵源
                    └──────────────────────────────────┘
```

传感器 PUF 不是来替代 SRAM PUF 的。它是在 SRAM PUF 下面、传感器上面那一层——一个目前空白的层。

## 六、应用场景分化

| 场景 | 传感器与处理器关系 | 适用技术 |
|:---|:---|:---|
| 手机/平板内部传感器 | 焊死在板上、物理不可接触 | SRAM PUF 足够 |
| 家电温度探头 | 通过接插件连接、可替换 | 传感器 PUF 有意义但不必须 |
| 工业 4-20mA 变送器 | 两线制、远端、可被物理接触 | **传感器 PUF 独有优势** |
| 汽车 BMS 温度探头 | LIN bus 连接、电池包内可接触 | **传感器 PUF 独有优势** |
| 变压器油温传感器 | 现场设备、远离控制室 | **传感器 PUF 独有优势** |
| 可插拔传感器模组 | 物理可替换、接口标准化 | **传感器 PUF 独有优势** |
