# V6.x 多模式瞬态采样功能说明

## 目标

本功能用于提高 128 点采样窗口内的物理信息密度，只回传原始时域数据，由 PC 端后续分析 raw、diff、FFT、edge FFT、周期谐波等特征。

核心思想不是简单增加采样次数，而是在固定 128 点窗口内组织不同上下电激励，让传感器暴露更多动态响应、过冲截断响应和周期耦合响应。

## 基本采样要求

- 每帧采样长度：128 点。
- 每帧包含 CH1、CH2 两个通道。
- UART 只需要回传 raw 数据，不需要同时回传 FPGA FFT 结果。
- 每帧必须标记采样模式名称、事务编号、传感器编号或采集批次信息。
- 所有模式采样前都应保证初始电源状态明确，避免上一帧残余状态污染当前帧。

## 采样模式

### MODE_FULL：完整基准模式

物理含义：完整上电响应和完整下电响应基本互不干扰，是质量控制和旧数据对照基准。

采样组织：
```
0 点：上电并开始采样
64 点：下电
128 点：结束采样
```

用途：估计完整 ON/OFF 响应、高/低稳态、DC 漂移，与旧 MODE=64 数据对齐。

### MODE_POS_CUT：正向极值截断模式

物理含义：在上电过冲峰值附近下电，让未完成的过冲过程强烈干扰下电过程。

采样组织：
```
0 点：上电并开始采样
约第 8 点：下电
128 点：结束采样
```

### MODE_NEG_CUT：负向极值截断模式

物理含义：先让传感器高稳态，然后下电；在负向过冲最低点附近重新上电。

采样组织：
```
0 点：下电并开始采样
约第 9 点：上电
128 点：结束采样
```

### MODE_EXTREMA_CYCLE：周期极值截断模式

物理含义：反复在上电过冲峰值和下电过冲谷值附近翻转电源，形成高信息密度的周期性强扰动。

采样组织：ON 8 点 → OFF 9 点 → ON 8 点 → ... 重复至 128 点

### MODE_FULL_CYCLE：周期完整基准模式

物理含义：在 128 点中放入多个较完整的上下电过程，与周期极值截断模式对照。

采样组织：
```
0 点：上电 → 32 点：下电 → 64 点：上电 → 96 点：下电 → 128 点：结束
```

## 推荐采集优先级

优先：MODE_FULL、MODE_POS_CUT、MODE_EXTREMA_CYCLE
次要：MODE_NEG_CUT、MODE_FULL_CYCLE

每传感器每模式至少 50 帧，EXTREMA_CYCLE 可优先采到 100 帧。

## 评价指标

- KNN / RF 识别准确率
- silhouette 系数
- intra/inter 距离比
- 最相似传感器对分离比（B2-6 vs B2-8）
