## 一、校准系统架构总览

本项目的校准系统不是通用ADC校准，而是**面向瞬态PUF身份认证的前端自校准方法**。其核心设计目标不是追求绝对电压精度，而是**在多通道、多传感器场景下建立统一的电压参考系，使不同传感器的瞬态响应在可比尺度上进行身份提取**。

### 1.1 三级校准流程（Zero-Reference-Gain）

```
State 0 (idle)        ->  State 1 (capturing)  ->  State 2 (computing)  ->  State 3 (done)
      ↓                      ↓                       ↓                      ↓
  等待用户按键         1024点平均采集            增益计算+存储            校准完成
  key1_press_pulse     calibration_capture       gain = target/span       calibration_valid=1
```

对每个启用的通道，依次执行：
1. **Zero点校准** (calibration_point_select=0): 采集零输入电压，记录 calibration_zero_measured[channel]
2. **Reference点校准** (calibration_point_select=1): 采集参考电压(2.5V=25000*100uV)，记录 calibration_ref_measured[channel]
3. **Gain计算** (state 2): gain_q = (TARGET << 16 + span/2) / span，定点Q16格式存储

### 1.2 关键参数

| 参数 | 值 | 含义 |
|:---|:---|:---|
| CALIBRATION_GAIN_SHIFT | 16 | 定点增益小数位宽 |
| CALIBRATION_TARGET_100UV | 25000 | 参考目标电压 = 2.5V |
| CALIBRATION_CHANNEL_MASK | 8'b0000_0011 | 仅CH1-CH2启用校准 |
| SAMPLE_COUNT | 1024 | 每次校准点采集样本数 |

### 1.3 校准公式（RTL实现）

顶层计算（transient_puf_test_top.v:958-962）:
  calibration_span_voltage = pending_ref_voltage_100uv - calibration_zero_measured[pending_gain_channel];
  calibration_gain_numerator = (CALIBRATION_TARGET_100UV <<< CALIBRATION_GAIN_SHIFT) + (span >>> 1);
  calibration_gain_computed = numerator / span;  // 带舍入的定点除法

应用校准（channel_voltage_calibration.v:24-46）:
  delta_voltage = uncalibrated_voltage - zero_voltage_measured;
  scaled_product = delta_voltage * gain_q;
  // 有符号舍入: + (1 << 15) for positive, - (1 << 15) for negative
  rounded_product = scaled_product +/- (1 << (GAIN_SHIFT - 1));
  rounded_correction = rounded_product >>> GAIN_SHIFT;
  // 钳位到 +/-50000 (+/-5V)
  calibrated_voltage = clamp(rounded_correction, -50000, 50000);

### 1.4 多通道选择性校准

通过 channel_mask + first_enabled_channel / has_next_enabled_channel / next_enabled_channel
自动遍历所有启用通道，跳过未启用通道。

### 1.5 校准与主状态机的耦合

- 校准状态机 (calibration_display_state 0-3) 独立于主状态机 (main_state)
- calibration_active 为高时，主状态机强制进入 ST_CALIBRATION
- 校准完成后 calibration_active 拉低，主状态机自动回到 ST_IDLE
- key2_press_pulse 可随时重启校准，或在校准中退出

## 二、专利点挖掘

### 专利点1: 面向瞬态身份认证的ADC前端三级自校准方法

- **一句话描述**: 在传感器瞬态PUF身份认证系统中，通过零偏校准-参考点校准-定点增益计算的三级流程，为每个通道建立独立的电压参考系，使不同传感器在统一尺度下的瞬态响应具备可比性。
- **技术效果**:
  - 消除ADC通道间零偏差异，避免同一传感器在不同通道上呈现不同身份
  - 消除ADC增益差异，使不同板卡/不同批次传感器的响应在相同物理尺度上比较
  - 定点Q16格式增益存储，无需浮点运算，适合FPGA资源受限场景
  - 每通道独立校准参数，支持多传感器并行身份提取
- **证据文件**:
  - rtl/transient_puf_test_top.v 第859-999行（校准状态机）
  - rtl/calibration_capture.v（1024点平均采集）
  - rtl/channel_voltage_calibration.v（定点增益应用+有符号舍入+钳位）
  - rtl/adc_voltage_scaler.v（ADC码值到100uV单位的线性映射）
- **与现有三件专利的关系**:
  - 与专利01（上下电瞬态响应身份提取）的关系：**强互补**。专利01描述如何提取身份，本专利点描述如何确保提取的尺度一致。可作为专利01的从属权利要求，也可独立成案。
  - 与专利02（SCALE_SCH挑战扫描）的关系：弱相关。校准发生在挑战扫描之前，是前端预处理。
  - 与专利03（多特征融合稳定比特量化）的关系：间接支撑。校准后的统一尺度有助于稳定比特的跨设备可比性。
- **需要补的实验**:
  - 同一传感器在校准前/后的类内/类间距离变化（证明校准提升身份提取稳定性）
  - 不同板卡（不同ADC零偏/增益）使用同一传感器，校准前后身份一致性对比
  - 校准参数随温度/时间的漂移跟踪
  - 校准失败场景（span=0或负值）的容错行为验证
- **新颖性风险**:
  - **中高**。通用ADC自校准（zero+gain two-point calibration）是成熟技术，大量存在于ADC芯片数据手册和工业校准流程中。
  - **规避方向**: 不要主张两点校准本身，而要主张将两点校准的结果**用于**瞬态PUF身份认证的前端统一尺度建立，强调校准目标不是绝对精度而是**身份可比性**，以及多通道选择性校准、定点Q16格式、与上下电瞬态采集流程的耦合。

### 专利点2: 基于通道掩码的多通道选择性自校准装置

- **一句话描述**: 通过可配置的通道掩码和自动遍历函数（first_enabled_channel / next_enabled_channel），在8通道ADC系统中对任意子集通道执行独立校准，跳过未使用通道以节省校准时间。
- **技术效果**:
  - 仅需修改 CALIBRATION_CHANNEL_MASK 即可适配不同传感器数量（1-8通道）
  - 自动遍历逻辑避免硬编码通道序列，RTL可综合为组合逻辑
  - 未启用通道不占用校准时间，提升系统启动效率
  - 每通道独立存储 zero_measured / ref_measured / gain_q / valid，互不干扰
- **证据文件**:
  - rtl/transient_puf_test_top.v 第36行（CALIBRATION_CHANNEL_MASK）
  - 第67-112行（first_enabled_channel / has_next_enabled_channel / next_enabled_channel 函数）
  - 第864行和第938-946行（自动遍历调用）
  - 第143-146行（每通道独立校准参数数组）
- **与现有三件专利的关系**:
  - 与专利01的关系：可作为从属权利要求，强调多通道传感器身份提取系统中的校准装置。
  - 与专利02/03的关系：弱相关。
- **需要补的实验**:
  - 不同 CHANNEL_MASK 配置下的校准时间对比
  - 8通道全启 vs 2通道启用的资源占用对比
  - 通道间校准参数差异统计（证明不同通道确实需要独立校准）
- **新颖性风险**:
  - **高**。通道掩码和自动遍历在多通道ADC系统中是常见设计模式，独立创造性弱。
  - **建议**: 不建议独立申请，作为专利点1或专利01的从属权利要求即可。

### 专利点3: 带双向舍入与饱和钳位的定点校准电压转换方法

- **一句话描述**: 在FPGA定点环境中，对校准后的电压值执行有符号双向舍入（正数向上、负数向下）和饱和钳位，防止定点运算中的截断误差和溢出导致身份特征畸变。
- **技术效果**:
  - 有符号舍入避免传统截断导致的系统性偏差
  - 饱和钳位将输出限制在 +/-50000（+/-5V），防止定点溢出产生错误身份特征
  - 全部使用定点运算，无需DSP浮点资源
  - 校准无效时（calibration_valid=0 或 gain_q=0）自动旁路校准，保证安全降级
- **证据文件**:
  - rtl/channel_voltage_calibration.v 第24-48行（完整校准应用逻辑）
  - 第33-37行（有符号舍入）
  - 第40-46行（饱和钳位）
  - 第28-29行（校准无效旁路逻辑）
- **与现有三件专利的关系**:
  - 与专利01的关系：可作为从属权利要求，强调在资源受限FPGA中实现高精度身份特征提取的定点运算方法。
  - 与专利02/03的关系：弱相关。
- **需要补的实验**:
  - 有符号舍入 vs 简单截断的量化误差对比（对身份特征距离的影响）
  - 饱和钳位触发频率统计（正常采集 vs 异常采集）
  - 校准旁路模式下身份提取性能降级幅度
- **新颖性风险**:
  - **中高**。定点舍入和饱和钳位是数字信号处理的基础技术。
  - **规避方向**: 强调在传感器瞬态身份认证场景下，定点运算误差会直接转化为身份特征误差，因此需要特定的舍入+钳位策略来保证认证准确率。
  - **建议**: 作为专利点1的从属权利要求，不独立申请。

### 专利点4: 校准状态机与瞬态采集状态机的互锁与优先级仲裁方法

- **一句话描述**: 在传感器身份认证系统中，校准状态机与主采集状态机通过 calibration_active 标志实现优先级仲裁：校准期间强制阻塞主状态机进入瞬态采集，校准完成后自动释放，且支持用户随时中断校准。
- **技术效果**:
  - 防止校准中误触发采集导致校准参数污染瞬态数据
  - 校准完成后自动进入空闲态，无需人工干预即可开始身份采集
  - key2_press_pulse 支持校准重启和强制退出，提升交互灵活性
  - 校准状态机4状态与主状态机14状态解耦，便于独立验证
- **证据文件**:
  - rtl/transient_puf_test_top.v 第1195-1214行（calibration_active 强制主状态机进入 ST_CALIBRATION）
  - 第859-999行（校准状态机完整逻辑）
  - 第887-914行（key2_press_pulse 重启/退出逻辑）
- **与现有三件专利的关系**:
  - 与专利01的关系：强互补。专利01描述身份提取流程，本专利点描述校准与采集的时序互锁机制。
  - 与专利02/03的关系：弱相关。
- **需要补的实验**:
  - 校准中触发采集的边界条件测试（验证互锁有效性）
  - 校准中断后系统恢复时间测量
  - 校准-采集切换过程中的数据完整性验证
- **新颖性风险**:
  - **高**。状态机优先级仲裁是通用数字设计技术。
  - **建议**: 不独立申请，作为系统实施细节写入专利01的说明书。

## 三、核心问题：面向瞬态身份认证的ADC前端自校准方法是否可独立成专利？

### 结论：可以，但需谨慎定位

| 维度 | 评估 |
|:---|:---|
| 技术效果独特性 | 中。通用ADC校准追求绝对精度，本方案追求**身份可比性**，目标不同。 |
| 与现有三件专利的耦合度 | 高。校准是专利01（身份提取）的前置步骤，天然适合作为从属。 |
| 独立创造性 | 中低。两点校准本身是成熟技术，新颖性在于**应用场景**和**与瞬态PUF流程的耦合**。 |
| 权利要求撰写难度 | 中高。需要巧妙区分通用ADC校准和用于传感器身份认证的校准。 |
| 实验补证工作量 | 中等。需要证明校准对身份提取性能有实质提升。 |

### 推荐策略

**方案A（推荐）：作为专利01的强从属权利要求**

在专利01（上下电瞬态响应传感器身份提取）中增加一组从属权利要求：

- 根据权利要求X所述的方法，其特征在于，在采集瞬态响应之前，
  对每个启用通道执行零偏校准和参考点校准，计算定点增益系数，
  并使用所述定点增益系数对采集的瞬态响应进行校准，以消除
  模数转换通道间的零偏差异和增益差异。
- 根据权利要求X所述的装置，其特征在于，还包括校准单元，
  所述校准单元包括：零偏采集模块、参考点采集模块、定点增益
  计算模块和校准应用模块；其中所述定点增益以Q16格式存储。
- 根据权利要求X所述的装置，其特征在于，所述校准单元通过
  通道掩码选择性地对多个通道中的部分通道执行校准。

**方案B（备选）：独立申请，但需强化差异化**

如果坚持独立申请，需要补充以下差异化要素：
1. 校准目标不是精度而是身份可比性 —— 实验证明同一传感器跨板卡/跨通道的身份一致性在校准后显著提升
2. 与瞬态采集的时序耦合 —— 校准作为每次上电前的必要步骤，校准参数随会话生效
3. 多传感器场景下的校准参数管理 —— 每个传感器绑定一组校准参数，更换传感器时自动重新校准
4. 定点Q16格式在身份特征距离计算中的优势 —— 证明定点校准后的特征距离分布优于浮点或未经校准的分布

### 风险清单

| 风险 | 等级 | 规避措施 |
|:---|:---:|:---|
| 被审查员认定为通用ADC两点校准 | 高 | 强调应用场景（瞬态PUF）、校准目标（身份可比性而非绝对精度）、与主状态机的耦合 |
| 实验数据不足以证明校准对身份提取的提升 | 中 | 补充校准前/后类内/类间距离对比实验 |
| 与专利01权利要求重叠 | 中 | 若作为从属则无此风险；若独立申请则需明确边界 |
| 定点运算细节被认定为公知常识 | 中 | 不将定点运算本身作为创新点，而是作为在资源受限场景下实现身份可比性的技术手段 |

## 五、建议与下一步

1. **不新增独立专利**，将校准系统作为专利01（上下电瞬态响应传感器身份提取）的强从属权利要求组。
2. **补充实验**：至少完成校准前/后身份一致性对比，证明校准对瞬态PUF身份提取有实质技术贡献。
3. **若未来发现校准参数本身具有传感器区分能力**（如不同传感器的校准gain/zero差异可区分身份），则重新评估独立申请价值。
4. **权利要求撰写时注意**：不要写一种ADC校准方法，要写一种用于传感器瞬态物理身份提取的前端校准方法。
