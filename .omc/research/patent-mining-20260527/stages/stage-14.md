## 1. 现有专利覆盖边界

| 专利 | 覆盖范围 | 未覆盖的CRP相关点 |
|:---|:---|:---|
| 01_上下电瞬态响应身份提取 | 物理激励、采集、特征提取、模板注册验证 | 无挑战参数、无双态共用挑战、无服务器端协议 |

- 响应如何与注册模板绑定验证
- 防重放攻击机制

## 2. 挖掘出的专利点

### 专利点-14A: ON/OFF双态共用挑战码字的绑定认证方法

**一句话描述**: 在传感器上电和下电瞬态采集过程中，通过硬件门控使上下电响应共用同一挑战码字，并将上下电响应对作为绑定对进行联合身份验证。

**技术效果**: 
- 上电时挑战计数器递增（advance_sch=1），下电时保持同一码字（advance_sch=0），确保ON/OFF响应对应同一挑战
- 防止攻击者分别伪造ON响应和OFF响应后拼接（因为不同挑战码字产生的响应不可互换）
- 将信息熵从单一状态的~2.3 bits提升到ON+OFF融合的~3.3 bits

**证据文件**:
- rtl/fft_256.v 第36-42行: SCALE_SCH counter映射逻辑
- rtl/fft_256.v 第146-151行: if (advance_sch) scale_counter <= scale_counter + 1
- rtl/transient_puf_test_top.v 第548-549行: assign fft_advance_sch = !fft_is_power_off
- rtl/transient_puf_test_top.v 第1246-1247行: ON阶段fft_is_power_off <= 1'b0，触发advance
- rtl/transient_puf_test_top.v 第1373行: OFF阶段fft_is_power_off <= 1'b1，保持当前sch
- doc/plans/CRP_SCALE_SCH_挑战响应方案.md 第1.3节: ON可区分5组，ON+OFF融合10/10唯一区分

**与现有专利关系**:
- 01主案: 作为从属权利要求——权利要求X所述的方法，其特征在于，上电响应和下电响应在同一挑战参数下成对采集
- 02挑战扫描: 补充实施例——共用挑战码字是响应图谱的一种特殊组织方式
- 03稳定比特: 可作为联合特征的输入来源

**需补实验**:
- ON/OFF共用挑战 vs 不同挑战的认证准确率对比
- 共用挑战下环境漂移（温度/压力）的稳定性验证
- 攻击场景：若ON和OFF使用不同挑战，拼接攻击成功率

**新颖性风险**: 中低。PUF中ON/OFF双态已有文献，但"通过硬件门控强制共用挑战码字并将响应对绑定"是具体实现层面的创新。关键在于强调硬件门控机制（advance_sch）和绑定验证逻辑，而非泛泛的双态采集。

### 专利点-14B: 基于定点FFT溢出行为的不可复现挑战-响应绑定

**一句话描述**: 利用FPGA定点FFT IP在溢出场景下的不可精确复现行为（连Xilinx官方C model也无法bit-accurate复现），将挑战参数映射到物理不可克隆的响应，使PC端软件无法模拟或预测响应。

**技术效果**:
- 10种PC模拟方法（含Xilinx C model）全部失败，最高相关性仅0.87
- Xilinx PG109官方文档明确承认：溢出场景下C model不bit-accurate
- 不同SCALE_SCH挑战→不同stage溢出模式→不同传感器产生差异化响应
- 形成"物理不可克隆 + 数字不可模拟"的双重安全边界

**证据文件**:
- doc/plans/CRP_SCALE_SCH_挑战响应方案.md 第3节: 为什么PC端无法精确复现
- 同文件第3.2节: Xilinx PG109原文引用——溢出时C model输出被忽略
- 同文件第3.1节表格: 10种方法全部失败
- rtl/fft_256.v 第41-42行: current_scale_sch映射逻辑，256个挑战值
- rtl/fft_256.v 第56行: s_axis_config_tdata配置SCALE_SCH

**与现有专利关系**:
- 01主案: 可作为独立权利要求的增强——权利要求X所述的频谱变换，其特征在于，所述变换采用定点运算并在数据溢出时产生wrap-around行为
- 02挑战扫描: 核心实施例支撑——不可复现性是挑战扫描响应图谱的物理基础
- 03稳定比特: 溢出行为的稳定性直接影响bit化可行性

**需补实验**:
- 不同FPGA芯片（同型号不同个体）的溢出行为一致性/差异性
- 温度变化对溢出模式的影响
- 更多SCALE_SCH值下的响应差异性量化

**新颖性风险**: 中。定点FFT溢出是已知现象，但"将其用于传感器身份认证的CRP绑定，并利用其不可复现性作为安全边界"是新的应用方向。需避免主张FFT溢出本身，而是主张"将定点非线性观测用于传感器瞬态身份图谱生成的系统化方法"。

### 专利点-14C: 服务器指定挑战子集的防重放认证协议

**一句话描述**: 认证服务器从预筛选的挑战子集中随机抽取一个或多个挑战参数下发给待验证设备，设备按指定挑战采集瞬态响应并返回，服务器将响应与注册模板中对应挑战的响应进行匹配，每次认证使用不同挑战组合以防止重放攻击。

**技术效果**:
- 注册阶段：采集全挑战图谱（256个SCALE_SCH），筛选最优子集（如Greedy Top-8）
- 验证阶段：服务器随机抽取挑战子集（如3-5个），设备只需采集指定挑战的响应
- 攻击者无法通过单次窃听获得完整模板（只获得部分挑战的响应）
- 挑战子集可动态更新（如定期轮换），被窃取的旧响应对失效

**证据文件**:
- PUF_dataTransFreq/scripts/analyze_sch_subset_selection.py: 挑战子集筛选算法（MI排序、贪婪选择）
- 同文件第212-238行: Greedy forward selection实现
- 同文件报告: Greedy Top-8 achieves 99%+ accuracy
- doc/plans/CRP_SCALE_SCH_挑战响应方案.md 第2.2节: CRP信息熵估算（5挑战→~10 bits）
- rtl/fft_256.v 第37行: 8-bit scale_counter支持256个挑战值

**与现有专利关系**:
- 01主案: 作为验证阶段的增强——权利要求X所述的验证阶段，其特征在于，认证端随机指定一个或多个挑战参数
- 02挑战扫描: 自然延伸——从"扫描全部"到"服务器指定子集验证"
- 03稳定比特: 若bit化成功，挑战子集可作用于bit串的不同位置

**需补实验**:
- 注册集/验证集分离后的误识率和拒真率（按挑战子集分离）
- 不同挑战数量（1/3/5/8/16）下的认证性能曲线
- 重放攻击模拟：窃听者记录一次认证响应后重放的成功率
- 挑战子集轮换策略（多久换一次、换多少）

**新颖性风险**: 中高。CRP防重放是经典PUF的常规做法，但本方案的区别在于：
1. 挑战对象是传感器瞬态响应（而非硅PUF的延迟/仲裁器）
2. 挑战参数是定点FFT配置（而非电压/温度等物理量）
3. 响应是频谱特征（而非bit串）
需强调"传感器 + 定点观测 + 频谱响应"的组合，而非泛泛的CRP。

### 专利点-14D: 带挑战元数据的二进制响应帧协议

**一句话描述**: 在设备向认证端回传的每一帧二进制响应数据中，嵌入当前挑战参数索引、挑战参数值、事务ID和电源状态标志，使认证端能够精确将响应对应到注册模板中的特定挑战-响应对。

**技术效果**:
- 二进制帧头包含：MAGIC(0xA5,0x5A)、协议版本、帧类型、txn_id[15:0]、is_power_off、payload_len、scale_sch_index[8:0]、scale_sch_value[11:0]
- 认证端无需猜测响应对应哪个挑战——元数据明确标识
- 事务ID支持请求-响应配对，防止帧乱序或丢失导致的匹配错误
- STATUS文本行也包含SCALE_SCH index和hex值，便于调试

**证据文件**:
- rtl/uart_feature_streamer.v 第1008-1044行: 二进制CH1帧发送逻辑
- 同文件第1012-1025行: 帧头结构——MAGIC0/1、PROTO_VERSION、FRAME_TYPE、txn_id、is_power_off、payload_len、scale_sch_index、scale_sch_value
- 同文件第276-334行: STATUS文本格式包含SCALE_SCH index和hex值
- rtl/transient_puf_test_top.v 第1126-1146行: UART请求队列机制，冻结per-frame元数据
- 同文件第1335-1338行: ON阶段将scale_sch_index/value存入uart_req
- 同文件第1456-1459行: OFF阶段同样存入（保持同一值）

**与现有专利关系**:
- 01主案: 作为数据输出格式的从属——权利要求X所述的方法，其特征在于，所述身份响应包含挑战参数标识
- 02挑战扫描: 支撑——响应图谱需要精确的挑战-响应对映射
- 03稳定比特: 若bit化，元数据可扩展为"不稳定bit位置指示"

**需补实验**:
- 帧格式在大量传输中的可靠性（丢帧率、误码率）
- 认证端解析帧并匹配模板的端到端延迟
- 事务ID冲突概率评估（16-bit txn_id在长时间运行中的回绕）

**新颖性风险**: 低。带元数据的二进制帧是通用通信协议设计，创造性较弱。建议不作为独立专利，而是作为系统权利要求中的"认证输出单元"组成部分，或写入01/02的从属权利要求。

### 专利点-14E: 挑战-响应模板的分挑战索引存储与动态更新

**一句话描述**: 注册模板不按传感器整体存储，而是按挑战参数分索引存储每个挑战对应的响应特征，支持认证时快速检索指定挑战的模板子集，并支持基于新采集数据对模板进行增量更新。

**技术效果**:
- 模板结构：Template[sensor_id][challenge_id] = {median_feature, variance, stability_score}
- 认证时只需加载服务器指定挑战的模板子集，无需加载完整256挑战图谱
- 支持模板老化检测：当某挑战的类内方差超过阈值时标记为"需重注册"
- 支持增量更新：新验证样本可与模板融合（如指数加权移动平均）

**证据文件**:
- PUF_dataTransFreq/scripts/analyze_500_baseline_and_drift.py: 模板生成逻辑（median templates per class）
- PUF_dataTransFreq/scripts/analyze_sch_subset_selection.py: 按sch分索引的特征构建（build_features(X, sch_indices)）
- doc/plans/CRP_SCALE_SCH_挑战响应方案.md 第2.2节: 多挑战信息熵估算

**与现有专利关系**:
- 01主案: 作为模板存储的增强——权利要求X所述的身份模板，其特征在于，按挑战参数分索引存储
- 02挑战扫描: 自然延伸——响应图谱的存储组织方式
- 03稳定比特: 稳定bit模板同样需要分挑战索引

**需补实验**:
- 分挑战模板存储的内存占用 vs 全图谱存储
- 模板增量更新后的认证准确率变化
- 模板老化检测的误报率（正常漂移 vs 真正老化）

**新颖性风险**: 中高。模板分索引存储是数据库/缓存的常规做法。创造性在于"传感器瞬态响应 + 挑战参数索引 + 稳定性评分"的组合。建议作为系统权利要求中的"身份模板生成单元"的一部分，不单独申请。

### 专利点-14F: 传感器瞬态CRP与硅PUF CRP的本质区别方法

**一句话描述**: 一种区别于经典硅PUF（如仲裁器PUF、环形振荡器PUF）的传感器身份认证方法，其挑战参数作用于数字观测配置而非物理结构，响应来自传感器模拟瞬态响应经定点变换后的频谱特征，且响应无法被软件精确复现。

**技术效果**:
- 经典硅PUF：挑战→物理结构（延迟线、RO对）→响应（bit）
- 本方案：挑战→数字观测配置（SCALE_SCH）→传感器物理瞬态→定点变换→响应（频谱/特征）
- 关键区别：
  1. 物理熵源是外部传感器（制造差异、寄生参数），而非芯片内部结构
  2. 挑战不修改物理结构，只修改观测方式
  3. 响应是连续频谱特征（可进一步量化），而非离散bit
  4. 响应不可被PC软件复现（定点溢出行为的不可模拟性）

**证据文件**:
- doc/plans/CRP_SCALE_SCH_挑战响应方案.md 全文: CRP方案设计
- rtl/fft_256.v: SCALE_SCH硬件实现
- rtl/transient_puf_test_top.v: 完整采集-变换-输出流程
- PUF_dataTransFreq/scripts/analyze_sch_subset_selection.py: 响应区分度验证

**与现有专利关系**:
- 01主案: 核心区别声明——在背景技术部分明确对比现有sensor PUF和SRAM PUF
- 02挑战扫描: 支撑——挑战作用于观测配置是区别于结构修改的关键
- 03稳定比特: 若bit化成功，需声明与经典硅PUF bit响应的区别

**需补实验**:
- 与经典PUF（如SRAM PUF、RO PUF）的性能对比（误识率、拒真率、熵/面积比）
- 传感器替换攻击：同一FPGA换不同传感器后的响应变化
- FPGA替换攻击：同一传感器换不同FPGA后的响应变化

**新颖性风险**: 低。这不是一个可独立申请的专利点，而是应在01/02/03的背景技术和权利要求中明确声明的"区别于现有技术"的论点。建议在每件专利的"背景技术"和"发明内容"部分写入此对比。

## 4. 关键实验缺口

1. **注册/验证分离实验**: 当前所有分析都是交叉验证，没有真正的注册集/验证集分离。必须补做：
   - 注册集：每个传感器前25次采集 → 建模板
   - 验证集：每个传感器后25次采集 → 测误识率/拒真率
   - 按挑战子集分离：注册用Top-8，验证时服务器随机抽3-5个

2. **重放攻击模拟**: 记录一次认证的全部UART帧，在后续认证中重放，测量服务器接受率。

3. **ON/OFF共用挑战 vs 分离挑战**: 对比两种策略在拼接攻击下的安全性。

4. **模板老化与更新**: 长时间运行（如1000次认证）后，模板是否需要更新、如何更新。

5. **跨FPGA一致性**: 同型号不同FPGA芯片的SCALE_SCH溢出行为是否一致（影响可移植性）。

## 5. 权利要求预埋建议

### 放入专利01（上下电瞬态响应）的从属权利要求

- 权利要求X所述的方法，其特征在于，所述上电响应和下电响应在同一挑战参数下成对采集，所述挑战参数为频谱变换配置参数。
- 权利要求X所述的方法，其特征在于，所述身份模板按挑战参数分索引存储，认证时根据指定挑战参数检索对应模板子集。

### 放入专利02（挑战扫描响应图谱）的从属权利要求

- 权利要求X所述的方法，其特征在于，所述定点频谱变换采用缩放调度参数控制各级蝶形运算的动态范围，在数据溢出时产生wrap-around行为，且所述wrap-around行为无法被软件精确复现。
- 权利要求X所述的方法，其特征在于，验证阶段由认证端从所述挑战参数子集中随机抽取一个或多个挑战参数下发给待验证端，待验证端按指定挑战参数采集响应并返回，认证端将所述响应与注册模板中对应挑战参数的响应进行匹配。
- 权利要求X所述的方法，其特征在于，所述上电频谱变换和下电频谱变换共用同一挑战参数，上下电响应对作为绑定对参与身份验证。

### 系统权利要求增强（适用于全部三件）

- 一种传感器身份认证系统，其特征在于，还包括：挑战配置单元，用于向定点频谱变换单元提供多个配置参数；响应绑定单元，用于将同一挑战参数下的上电响应和下电响应作为绑定对；认证判决单元，用于根据认证端指定的挑战子集从注册模板中检索对应响应并进行匹配。
