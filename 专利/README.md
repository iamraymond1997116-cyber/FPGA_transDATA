# 专利草案工作区

本目录用于整理本项目的专利申请思路、技术大纲、权利要求骨架和后续实验补证清单。

当前文件仅为技术草案，不构成法律意见。正式提交前需要由专利代理人结合检索结果重写权利要求，并确认新颖性、创造性、实用性和申请主体。

## 当前申请族建议

建议优先形成一个主申请，再根据检索结果拆分分案或后续申请。

| 优先级 | 方向 | 建议定位 | 当前目录 |
|:---:|:---|:---|:---|
| P0 | 受控上下电瞬态响应提取传感器身份 | 主案，最像完整技术方案 | `01_上下电瞬态响应传感器身份提取/` |
| P1 | `SCALE_SCH` 挑战扫描生成稳定响应图谱 | 可作为主案从属，也可拆第二案 | `02_SCALE_SCH挑战扫描响应图谱生成/` |
| P2 | 多特征融合与稳定比特量化 | 更适合作为从属或后续改良案 | `03_多特征融合稳定比特量化/` |

## 撰写原则

1. 不把算法孤立地写成“分类方法”，而是写成“硬件激励、采集链、信号处理和身份认证共同构成的技术方案”。
2. 不把 `SCALE_SCH` 写成新的物理熵源，而是写成“数字观测挑战参数”或“定点非线性观测算子”。
3. 不直接主张“阈值量化成 bit”本身可专利，而要强调多条件稳定性筛选、上下电一致性、硬件采集约束和抗漂移技术效果。
4. 每个申请至少准备方法、装置/系统、计算机可读存储介质或程序产品三类权利要求骨架。

## 还值得考虑的候选专利点

这些点暂时先不拆独立文件夹，建议后续按实验结果决定是否单独申请。

| 候选点 | 建议优先级 | 适合写法 | 风险 |
|:---|:---:|:---|:---|
| 可编程电源激励序列/PWM 软启动挑战 | 高 | 通过上升沿斜率、电压台阶、占空比、放电路径等形成物理挑战，生成 CRP | 需要补硬件验证 |
| 上下电双态一致性与环境漂移自适应认证 | 高 | 用 ON/OFF 双响应、温度/压力漂移模型和异常阈值实现稳健认证 | 容易被写成普通异常检测，需绑定传感器瞬态 |
| 多通道差分/比值抗共模指纹 | 中高 | 利用 CH1/CH2/多通道同步响应的差分、比值、相位或能量比例抑制共模噪声 | 需要证明优于单通道 |
| 采集质量门控与离群样本重采样 | 中 | 在采集端或分析端识别接触不良、错标、环境异常，避免污染注册模板 | 独立创造性可能偏弱 |
| FPGA 定点溢出/截断行为作为可控非线性观测器 | 中 | 利用固定点 FFT 内部动态范围控制，将传感器微小波形差异映射成可区分频谱图样 | 需要避开“IP 已知行为”的风险 |
| 频谱段压缩特征的边缘端认证 | 中 | 例如 Seg-Max 4 维、低存储模板、轻量比对，强调资源受限 FPGA/MCU 部署 | 可能更适合从属权利要求 |
| 注册-复验-重挑战协议 | 中 | 注册阶段筛选稳定挑战，验证阶段随机抽取挑战组合，降低重放风险 | PUF 协议现有技术较多 |

## 需要尽快补的证据

- 10 个以上传感器，至少 50 次重复采集的类内/类间统计。
- 上电、下电、ON/OFF 联合特征对比。
- `SCALE_SCH` 扫描前后的区分度、重复性和抗漂移对比。
- 温度、压力、时间漂移下的误识率、拒真率和重认证成功率。
- 连续特征认证与 bit 化认证的差距，尤其是 HD gap 是否转正。
- 资源占用：FPGA 逻辑、BRAM、采集时间、UART 数据量、模板存储量。
