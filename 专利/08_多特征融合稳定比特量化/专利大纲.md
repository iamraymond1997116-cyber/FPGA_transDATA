# 专利大纲：多特征融合与稳定比特量化

## 拟定名称

一种面向传感器瞬态物理指纹的多域特征融合与稳定比特生成方法、装置及系统

## 建议定位

第三优先级。当前连续特征识别效果较好，但直接 bit 化仍存在 HD gap 为负的问题。因此本方向不建议作为第一件主案，而应作为主案的从属方案或待实验成熟后的改良申请。

## 要解决的技术问题

连续特征可以较好地区分传感器，但在安全认证、密钥派生或低存储模板场景中，需要稳定、可重复、误码率低的离散响应或 bit 串。普通阈值量化容易受到环境漂移、采集噪声和类内波动影响，导致类内汉明距离大于类间汉明距离。

本方案要解决的问题是：如何从传感器上下电瞬态的多域特征中筛选稳定特征，并生成可用于认证或密钥派生的离散响应。

## 核心发明点

1. 同时使用时域、频域、峰位、段能量、多通道差分、上下电一致性和挑战扫描特征。
2. 注册阶段根据类内稳定性和类间分离度筛选可量化特征。
3. 对不稳定区间设置保护带，避免把边界样本强行量化。
4. 可结合 helper data、纠错码或重采样机制提高验证稳定性。
5. 输出可以是 bit 串、离散码字、半离散区间码或连续模板与离散辅助码的组合。

## 技术方案流程

### 特征候选生成

1. 采集传感器上电和/或下电瞬态响应。
2. 生成多个候选特征，包括：
   - 时域特征：上升/下降时间、稳定点、斜率、面积、过冲；
   - 频域特征：频段最大值、段能量、峰位、峰间距、谱矩；
   - 多通道特征：差分、比值、相关性；
   - 挑战特征：不同 `SCALE_SCH` 或等效配置下的响应；
   - 联合特征：ON/OFF 相对关系、温压漂移一致性。

### 稳定性筛选

1. 对每个候选特征计算类内波动。
2. 对不同传感器之间计算类间距离。
3. 选择满足稳定性裕量的特征。
4. 对边界区域设置保护带，保护带内的特征不用于 bit 生成，或触发重采样。

### 量化与认证

1. 对筛选后的特征进行二值或多值量化。
2. 生成 bit 串、区间码或组合响应。
3. 认证时比较汉明距离、编辑距离、码字距离或结合纠错码恢复。
4. 当不稳定 bit 数量超过阈值时拒绝认证或重新采集。

## 独立权利要求骨架

### 方法权利要求

一种传感器物理指纹比特生成方法，包括：

1. 获取传感器在受控电源瞬态过程中的多域响应特征；
2. 对所述多域响应特征进行类内稳定性和/或类间分离度评估；
3. 从所述多域响应特征中筛选满足稳定性条件的特征集合；
4. 对所述特征集合进行带保护区间的量化，生成离散身份响应；
5. 根据所述离散身份响应进行身份认证或密钥生成。

### 系统权利要求

一种稳定比特生成系统，包括特征生成单元、稳定性评估单元、特征筛选单元、保护带量化单元和认证输出单元。

## 从属权利要求方向

- 稳定性条件包括类内方差小于阈值、类间距离大于阈值或二者的比值满足阈值。
- 稳定性条件还包括跨条件变异系数（CV）：在温度/压力/时间漂移条件下，特征值的标准差与均值之比低于阈值。
- 保护带根据注册样本的均值、方差、温度漂移或压力漂移动态确定。
- 保护带宽度与类内方差成正比，类内方差大的特征分配更宽的保护带。
- 量化结果包括二值 bit、多值码字、区间编号或峰位排序码。
- 对 ON 响应和 OFF 响应分别量化后进行一致性校验：两个方向生成的bit串进行逐位比对，不一致位数超过阈值时拒绝认证。
- 对不同挑战参数（SCALE_SCH）下的响应分别量化后进行投票或纠错，多挑战投票一致时才接受该bit。
- 当不稳定 bit 数量超过阈值时，拒绝认证或触发基于辅助模板（helper data）的纠错恢复。
- 使用 helper data 指示不稳定 bit、保护带位置或纠错辅助信息。
- 所述不稳定 bit 数量阈值根据注册阶段的类内汉明距离分布动态确定。

## 可用实验支撑

- 当前报告显示连续特征如 Seg-Max、LDA、ratios、moments 具有较好区分度。
- 当前直接 bit 化存在负 HD gap，说明该方向的技术问题真实存在。
- ON 峰位差异、OFF 连续幅度差异、多通道差异均可作为候选特征来源。

## 需要补的实验

- 找到至少一种 HD gap 转正的 bit 生成策略。
- 给出稳定 bit 数量、误码率、拒真率、误识率。
- 验证 helper data 或保护带是否显著降低类内汉明距离。
- 验证温度、压力、时间漂移下 bit 串稳定性。
- 与简单 median threshold、Top-k peak code 等基线方法对比。

## 风险与规避

- 稳定 bit、helper data、ECC、fuzzy extractor 的现有技术较多，独立创造性风险高。
- 不建议把”阈值量化”作为核心创新。应强调量化对象来自受控瞬态、多通道、ON/OFF 和挑战扫描的联合物理响应。
- **在 HD gap 转正之前，不建议把该方向作为主申请。**当前直接bit化存在负HD gap（类内汉明距离 > 类间汉明距离），该负gap意味着bit化后同一传感器的不同采集比不同传感器之间的响应更不相似——即bit化特征不可用于认证。
- 若必须在HD gap转正前申请，建议作为专利01或专利04的从属权利要求预埋，保护”带保护区间的量化方法”和”多域特征融合生成bit串的流程”。

## 关联专利

- **专利01**（上下电瞬态响应）：本专利的物理特征来源。
- **专利04**（SCALE_SCH挑战扫描）：本专利多挑战投票的特征来源。
- **专利03**（定点FFT溢出特征放大器）：本专利中频域特征候选的增强来源。
