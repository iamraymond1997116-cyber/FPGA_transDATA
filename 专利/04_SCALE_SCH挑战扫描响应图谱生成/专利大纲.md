# 专利大纲：基于 SCALE_SCH 挑战扫描的响应图谱生成

## 拟定名称

一种基于频谱变换配置挑战扫描的传感器瞬态身份图谱生成方法、装置及系统

## 建议定位

第二申请或主案的重要从属。该方向保护的是“对同一瞬态响应施加不同数字观测配置，得到多视角响应图谱，并从中筛选稳定、可区分身份特征”的方法。

`SCALE_SCH` 在本项目中应表述为数字观测挑战参数，而不是新的物理熵源。它改变定点 FFT 内部动态范围和中间结果行为，从而改变传感器瞬态响应在频域中的可观测图样。

## 要解决的技术问题

单一频谱配置下，传感器瞬态差异可能不够稳定或不够易分离。需要一种可重复、可配置的观测机制，将同一物理瞬态响应映射到多个响应视角中，从而筛选出区分度高、重复性好的身份图谱。

## 核心发明点

1. 将频谱变换配置参数作为挑战输入，依次作用于同一类传感器瞬态响应。
2. 对每个挑战参数采集或计算上电、下电、多通道频谱响应。
3. 构建包含挑战参数、通道、上下电状态和频率 bin 的多维响应图谱。
4. 基于类内稳定性、类间分离度、峰位差异、段能量差异等指标筛选挑战子集。
5. 用筛选后的挑战响应图谱生成传感器身份模板或认证响应。

## 技术方案流程

### 挑战扫描阶段

1. 设定一组频谱变换配置参数，例如 0 至 255 的 `SCALE_SCH` 码字。
2. 对每个配置参数执行一次或多次受控上电/下电采集。
3. 在 FPGA 中以相应配置完成定点 FFT 或等效频谱变换。
4. 记录每个配置参数对应的 `SPECTRUM_CH1`、`SPECTRUM_CH2`、`OFF_SPECTRUM_CH1`、`OFF_SPECTRUM_CH2`。
5. 得到响应图谱：`R(challenge, power_state, channel, bin)`。

### 挑战筛选阶段

1. 对每个挑战参数计算类内方差、类间距离、峰位分布差异或 JS 散度。
2. 选择满足稳定性和可分离性条件的挑战参数。
3. 可进一步压缩为少量频段、峰位集合、Seg-Max 特征或 LDA 投影。
4. 生成挑战响应模板。

### 验证阶段

1. 服务器或认证端指定一个或多个挑战参数。
2. 待验证端按指定挑战参数采集瞬态响应并生成响应。
3. 认证端比较挑战响应与注册模板，完成身份判定。

## 独立权利要求骨架

### 方法权利要求

一种传感器身份图谱生成方法，包括：

1. 设置多个频谱变换配置参数作为挑战参数；
2. 在每个挑战参数下，对传感器的受控电源瞬态响应进行频谱变换；
3. 获取与所述挑战参数对应的频域响应；
4. 根据多个挑战参数及其对应频域响应构建传感器身份响应图谱；
5. 根据所述身份响应图谱生成传感器身份模板或认证响应。

### 系统权利要求

一种传感器身份图谱生成系统，包括挑战配置单元、瞬态采集单元、定点频谱变换单元、响应图谱构建单元和身份模板生成单元；其中挑战配置单元用于向定点频谱变换单元提供多个配置参数。

## 从属权利要求方向

### 挑战参数与配置
- 所述频谱变换配置参数包括 FFT 缩放调度、定点动态范围控制、舍入方式、截断方式或等效非线性观测配置。
- 所述频谱变换为定点FFT，所述定点FFT在特定缩放配置（如SCALE_SCH=0）下产生中间结果溢出，所述溢出将传感器微小幅值差异映射为频谱中确定性的图样差异（参见专利03）。

### 双态挑战同步
- 上电响应和下电响应在同一挑战参数下成对采集。
- 使用 `advance_sch` 或等效门控信号控制挑战码计数器的递增行为：上电采集时递增挑战码，下电采集时保持挑战码不变，使上下电共用同一挑战码序列（参见专利05）。
- 所述门控信号由主状态机中的上下电状态标记经逻辑取反产生：`advance_sch = !is_power_off`。

### 挑战筛选
- 挑战筛选指标包括类间距离、类内方差、JS散度、峰位稳定度、段能量差异或认证准确率。
- 挑战筛选还包括：跨环境条件（温度/压力/时间漂移）下对各挑战参数的响应稳定性评估，选择对环境变化最不敏感的挑战参数子集。
- 挑战子集选择采用贪婪算法或互信息排序，筛选后子集（如Top-8 SCH）可达到与全量256 SCH相当的区分度。
- 响应图谱可压缩为若干频段最大值、峰位分布、峰间距签名或低维LDA投影。

### 验证协议
- 验证阶段服务器随机抽取挑战子集以形成challenge-response认证，防止重放攻击。
- 认证服务器可动态选择挑战参数子集，客户端需在指定挑战参数下完成采集和响应生成。

## 可用实验支撑

- 当前固件支持 256 个 `scale_sch` 码字轮换。
- V5.6 已实现 `advance_sch` gating，使 ON/OFF 共用同一 code word。
- 已有 10 传感器、500 样本的 `256 sch x 4 line_type x 128 bin` 数据。
- 报告显示 ON 峰位差异在部分 `sch` 下更强，`sch=128/129` 等具有较高差异度。
- OFF `sch=0` 的 Seg-Max 4 维特征具有较高工程可用性。

## 需要补的实验

- 选定 challenge 子集后，注册/验证分离实验。
- 不同 challenge 数量下的认证性能曲线。
- ON/OFF 共 challenge 与不同 challenge 的对比。
- 环境漂移下 challenge 子集是否仍稳定。
- 与单一 FFT 配置相比，挑战扫描是否提升区分度或降低模板大小。

## 风险与规避

- PUF challenge-response 现有技术很多，应避免只写”给挑战、得响应”的泛泛方案。
- `SCALE_SCH` 是 FFT IP 的配置参数，不应主张参数本身新颖；应主张其在传感器瞬态身份图谱生成中的特定用途和系统流程。
- 若把该方向写成”物理梳齿放大”，容易被质疑机制不清。建议写成”定点非线性观测增强”或”多配置响应映射”。
- 权利要求中不直接出现”Xilinx”或”SCALE_SCH”商标名称，使用上位表述”定点频谱变换器的缩放调度参数”。
- 需补充挑战子集vs单一配置的对比实验数据，证明挑战扫描相对于简单单次FFT具有实质区分度提升。
